前后端分离项目本地HPPS快速解决方案

发表于 2023-06-21 分类于 Dev 阅读次数：

在本地开发中，有时候我们经常需要模拟https环境，比如需要获取GPS定位、摄像头权限等。在传统的解决方案中，我们需要使用自签名证书，然后在http server中使用它。

这些步骤需要一系列繁琐的openssl命令生成，而且需要频繁上传到支持https的服务器环境测试，非常地不方便！！！

下面介绍一种更加简单友好的方式生成本地https证书，并且信任自签CA的方案——mkcert

安装mkcert

  • 安装命令：npm install -g mkcert
  • 判断是否安装成功，输入命令：mkcert --version，如果能看到版本号，说明安装成功

生成CA相关证书

  • 新建目录，准备存放证书。如：D:\Users\wangh\Dev\VueCerts
  • 生成一个ca证书，命令mkcert create-ca

​ 生成之后会看到一个ca.crt和ca.key文件

使用 mkcert help create-ca 查看更多命令选项：

  • 再生成cert证书，命令mkcert create-cert

​ 会生成cert.crt和cert.key文件。可加上 –domains 参数，指定 ip 或 域名

使用 mkcert help create-cert 查看更多命令选项：

到此，生成证书完成，共有四个文件：ca.crt、ca.key、cert.crt、cert.key。

安装CA根证书

需要将刚刚生成的ca.crt安装到电脑受信任的根证书中

windows安装方式

  • 直接双击刚刚生成的ca.crt文件，弹出的界面中选择安装证书；

  • 下一步弹窗中，选择“当前用户”或“本地计算机”均可，选择“将所有的证书都放入下列存储”，并且选择为“受信任的根证书颁发机构”；

  • 后面，一路Next，点击“是”即可，到此证书安装成功。

如果你需要在局域网的其他机器上通过https访问应用，请把 ca.crt 文件复制到目标机器上同上安装即可。

后端Springboot配置

前端配置了https，如果是前后端分离的项目，后端api接口应用也需要配置https。

这里，我们不直接配置springboot的ssl。取而代之使用nginx反向代理转发请求。

nginx的下载安装过程省略。我己安装好 nginx-1.20.2。路径为

D:\tools\nginx-1.20.2\

安放证书文件

将mkcert生成的证书文件拷贝到目录，目录可任意。我将其放在D:\tools\nginx-1.20.2\www\ssl，并将它们改名。改名不是必需的，我这样做，只是为了将来可能存放多个站点的证书文件，以示区分。

D:\tools\nginx-1.20.2\www\ssl
